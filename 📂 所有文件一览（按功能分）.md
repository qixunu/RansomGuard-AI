## 🔄 完整实现流程（从上传文件到出结果）

整个产品的核心思想是：**先用 6 个传统安全特征给文件打恶意倾向分，再让大模型对这 6 个分数做"解释性融合"**。

### 流程 1：上传单个 PE 文件（最常用）

```
用户浏览器（index.html + app.js）
        │
        │  上传 exe/dll 文件 + 选择 provider + 填 Key
        ▼
 app.py 的 /api/scan 路由
        │
        │  ① 读文件 bytes → 调用 features.py
        ▼
 features.py 的 extract_all() / extract_info()
        │  计算 6 个维度：
        │    CFG  控制流复杂度（间接启发式）
        │    EH   字节熵直方图（衡量混淆/加密程度）
        │    GI   灰度图纹理复杂度
        │    OP   操作码频率（REP/STOS/MOVS 等可疑指令）
        │    PF   PE 结构特征（节区、导入表危险函数、可写+可执行等）
        │    RB   原始字节分布
        │  每个维度输出一个 0~1 的 float
        ▼
 detector.py 的 detect_sample()
        │
        │  ② 规则快速检查：
        │     - 所有模型 ≤ 0.15 → 直接判良性
        │     - ≥5 个模型 ≥ 0.85 → 直接判勒索
        │     - 明显分歧 → 交给融合引擎
        │
        │  ③ 加权融合 + 几何均值融合（本地兜底）
        │     ensemble_prob = (weighted + geometric) / 2
        │
        │  ④ LLM 推理（如果用户启用）：
        │     └→ 调用 llm_engine.call(6模型分数, 文件hash)
        │           │
        │           ▼  llm_engine.py 的 _call_provider()
        │                按 provider 分发到对应 _call_*：
        │                  _call_openai      → api.openai.com
        │                  _call_deepseek    → api.deepseek.com
        │                  _call_anthropic   → api.anthropic.com
        │                  _call_dashscope   → dashscope.aliyuncs.com（通义千问）
        │                  _call_local_http  → 自定义 URL（ollama/vllm）
        │                  mock              → 本地规则生成伪 LLM 回答
        │                所有真实 API 都返回 JSON
        │                _extract_json() 解析出 final_decision / reason /
        │                                  ensemble_probability / suggestion
        │           │
        │           ▼  失败处理：
        │                - 用户配置错误（Key/模型/路径）→ 抛错，不回退
        │                - 网络/服务端问题 → 回退本地加权融合
        │
        │  ⑤ 结果融合：
        │     LLM 概率 和 本地加权概率 做加权（LLM 分歧大 → LLM 权重高）
        │     生成 final_decision / reasoning / suggestion / 误报风险评估
        │
        ▼  返回 DetectionResult（含 verdict_source、llm_verdict 等）
 app.py
        │
        ▼
  浏览器（app.js 渲染）
        显示：综合恶意概率 %、风险等级、判决来源（llm+ensemble / ensemble / rule）、
             各模型评分条、LLM 推理原文、处置建议
```

### 流程 2：批量检测 CSV

```
用户上传 CSV（需包含 hash, CFG, EH, GI, OP, PF, RB 列）
        │
        ▼  app.py /api/batch
        │
        ▼  detector.py detect_from_csv()
             对每一行循环调用 detect_sample()（可选走 LLM）
             有 label 列时计算 Accuracy / Precision / Recall / F1
             返回完整结果 + 下载链接
```

### 流程 3：加载示例数据（/api/demo）

```
一键加载 data/sample_confidence.csv（1000 条预计算好的置信度）
→ detector.py 批量检测
→ 有 sample_wild_label.csv 时自动对比标签，输出评估指标
→ 前端展示表格，支持下载完整结果 CSV
```

## 🧠 关键设计点

1. **"可解释" 是核心**：不仅告诉你"这个文件是不是勒索"，还要给你：
   - 6 个模型各自的分数（**为什么可疑**）
   - LLM 的推理原文（**它看到了什么模式**）
   - 判决来源标识（`rule` / `ensemble` / `llm+ensemble`，让你知道结论从哪来）
   - 误报风险等级（帮助安全运营做二次核验）

2. **LLM 是"增强"而非"唯一来源"**：本地加权融合永远是兜底，断网/额度耗尽也能工作。

3. **特征提取是启发式实现**（见 `features.py`）：为了开箱即用不依赖反汇编引擎，CFG/GI/OP 等维度用统计方法近似计算。如果你有 pefile/capstone/IDA，可在 `features.py` 里替换为更精确的实现。

4. **config.json 是"全局旋钮"**：修改阈值、provider、重试次数等都在这里调整，无需改代码。
